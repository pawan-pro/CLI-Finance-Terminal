# Context Snapshot: Quantwater Terminal Build (Hackathon)
**Current Date:** January 19, 2026  
**Objective:** Build an institutional-grade CLI "AI Research Copilot" that analyzes proprietary market data, bond proxies, and economic events using Google Gemini 1.5 Flash.

## 1. Core Architecture (The Data Pipeline)
*   **Producers (Multi-Source):** 
    *   **MetaTrader 5 (MT5):** Exports M15 bar data for 50+ individual symbols (FX, Indices, Metals, Equities). Features an aggressive sync loop to force-update broker caches.
    *   **Twelve Data (T12):** (New) Provides high-frequency (M15) **Bond Proxies** (SHY, IEF, TLT) and **Sector ETFs** to replace stale, daily macro sources.
    *   **Google News RSS:** Provides real-time headlines for the "News Agent" to explain idiosyncratic price shocks.
*   **Bridge & Pathing:** Enforced **Forced Absolute Pathing** across all scripts to prevent database siloing. All components point to a single master DuckDB file.
*   **Silver Layer (DuckDB):** 
    *   `fetch_bonds.py`: Ingests T12 bond price action into `m15_bars` with `.px` suffix.
    *   `ingest_calendar.py`: Normalizes macro events (Actual vs. Forecast) to UTC.
*   **Analytics Brain:** A SQL engine (`metrics_engine.py`) featuring **Institutional Sector Mapping**. Assets are classified via symbol suffixes (`.px` for Bonds, `.sd` for FX, `.lv` for Crypto) to determine market hierarchy.
*   **AI Copilot:** A Gemini 1.5 Flash integration (`research_copilot.py`) featuring **Macro Regime Reasoning**вЂ”using bond price ratios (10Y/2Y) to detect yield curve steepening/normalization.

**Product Requirements Document: Financial News Agent Module for CLI Investment Terminal**

***

**Objective:**
Add a context-aware financial news section to daily investment reports generated by the CLI finance terminal, leveraging Google’s Agent Development Kit (ADK) with zero/low-cost deployment.

***

**Scope & Features:**
- **Asset Extraction:** Parse tickers/topics from daily investment report (HTML or structured export).
- **News Agent:** Build an ADK agent to fetch, filter, and summarize financial news headlines and stories relevant to extracted assets.
  - Source headlines from free-tier news APIs/web scraping (Moneycontrol, Bloomberg, Reuters, CNBC, etc.).
  - Use keyword matching and LLM-driven relevance filtering.
- **Summary Generator:** Use ADK workflow agents to produce concise news digests with optional sentiment tagging (positive/negative/neutral impact).
- **Report Integration:** Append the summarized news section to the final daily HTML report.
  - Option to distribute separately (Slack/email/chatbot integrations).
- **Configuration:** Modular codebase for future extension (new sources, new asset types, new output formats).
- **Deployment:** Run agent workflow locally and optionally containerize for free-tier cloud deployment (Google Cloud, AWS).
- **Cost Management:** Remain within free-tier usage for agents, compute, and external APIs.

***

**Target Users:**
- Investment analysts, asset managers, and Quantwater Tech clients using the CLI terminal for daily portfolio and market analysis.

***

**User Journey:**
1. **Data Fetch:** MT5 EA stores investment data; Python script SSHs data locally and generates daily report.
2. **Agent Workflow Trigger:** Post-report, ADK agent parses report, runs news workflow.
3. **News Digest:** Agent fetches headlines, filters for asset relevance, summarizes, and tags.
4. **Report Delivery:** News section is appended; user reviews/finds actionable headlines.

***

**Success Metrics:**
- Number of news digests delivered daily.
- Relevance and accuracy of news (user feedback).
- Workflow operates fully within free tier; no recurring costs.

***

**Technical Constraints:**
- All modules must run on open-source/free tools or within cloud free-tier usage.
- APIs or scraping workflows must abide by licensing and fair use.

***

**Milestones:**
1. Asset extraction and parsing (Week 1)
2. Build and test ADK news agent (Week 2)
3. Summary and sentiment module integration (Week 2)
4. Complete daily report integration, local deployment (Week 3)
5. Optionally test container/cloud deployment (Week 4)
